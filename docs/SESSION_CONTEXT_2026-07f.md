## Находка №1: rf в Sharpe — где вычитать, где нет

**Дыра (справедливая критика Александра):** Sharpe считался без вычета
rf. Excess низковольного комбо ≈ 0, а gross ~1.1-1.5 вводил в
заблуждение.

**Но при попытке вычесть rf везде вылезло:**
- Excess-Sharpe НЕ инвариантен к масштабу позиции:
  (kμ − rf)/(kσ) = μ/σ − rf/(kσ). Чем ниже реализованная вола, тем
  сильнее rf бьёт. Вычет rf из отдельной ноги (NG solo, вола ~3%) дал
  Sharpe −2.34 при плюсовом компаунде — бессмыслица.
- Одиночный инструмент несёт долю риск-бюджета (вола 2-4%, не 15%),
  а обеспечение зарабatывает rf ОДИН раз на весь счёт, не по разу на
  каждый из 12 активов.

**Решение (реализовано):**
- rf вычитается ТОЛЬКО на уровне полностью проинвестированного
  капитала (портфель/комбо), НЕ на уровне ноги или актива.
- Для фьючерсного счёта вычитать rf из P&L стратегии — ДВОЙНОЙ СЧЁТ:
  обеспечение в T-bills зарабатывает rf само -> excess счёта над
  T-bills = чистый P&L стратегии -> excess-Sharpe фьючерсного счёта =
  GROSS Sharpe. Честный ответ Александру: Sharpe комбо +1.34,
  доходность счёта = rf + стратегия = 4.5% + 1.8% = 6.3% против
  bonds 4.5%.
- Сравнение стратегий между собой — только на gross (bootstrap rf=0):
  rf-член в РАЗНОСТИ Sharpe при равной воле сокращается, но при разной
  воле загрязняет. Правильный арбитр качества сигнала — gross.

**Что в коде (6 файлов):** core/engine.py и engine_portfolio.py —

diagnostics/yearly.py — rf-параметр. diagnostics/bootstrap.py —

## Находка №3 (ГЛАВНАЯ): карта «актив × стратегия»

Новый инструмент: diagnostics/instrument_contribution.py +
runners/run_instrument_contribution.py. Для каждого актива:
индивидуальные return/DD/Sharpe + leave-one-out дельта портфельного
Sharpe (выкинули -> лучше/хуже). Sharpe gross (масштабо-инвариантен).

| GC    | −0.16 держ | −0.21| −0.12   | −0.09  | −0.07   | держать везде |

(+ = балласт, − = держать. Полные прогоны в чате сессии.)

**Прочтение:**
- CL, GC, ZS — трендовые: держать в тренде, балласт в MR-стопах.
- NG — реверсионный: балласт в тренде, ЛУЧШИЙ держатель в mr_kelt
  (−0.17) и хорош в kama (solo +0.50). Не «плохой газ» — не-трендовый.
- PL — балласт во ВСЕХ трендовых (donchian/kama/hurst/donch_vol),
  нейтрал в MR. Структурно не-трендовый.
- ZL, PA — балласт в MR со стопами, но не в тренде и не в mr_confirm.
- GC, SI — держать почти везде (сильные сами).

**Это эмпирическое обоснование Hurst-аллокатора:** направлять актив в
ту ногу, чьё свойство он проявляет, вместо гонки всех через всех.
NG/PL не удалять — переводить в правильную ногу.
