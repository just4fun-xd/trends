# Каталог стратегий и архитектура портфеля

Документ отвечает на три вопроса: (1) как устроен алгоритм от сигнала до
плеча, (2) что такое каждый рабочий sleeve, (3) полная таблица рабочих и
альтернативных стратегий с числами.

---

## Часть 1. Как работает алгоритм — 6 уровней

Портфель собирается снизу вверх. Каждый уровень — отдельный слой,
который можно менять независимо.

### Уровень 1. Сигнал (на одном инструменте)
Стратегия смотрит на цену одного инструмента (нефть, золото, Apple) и
выдаёт **позицию от 0 до 1**: 0 = вне рынка, 1 = полный лонг.
- Тренд (Дончиан, EMA): «покупай, когда цена пробила верх недавнего
  диапазона» — ставка на продолжение движения.
- Возврат к среднему (MR): «покупай, когда цена ушла слишком низко от
  средней» — ставка на отскок.

### Уровень 2. Vol-targeting (сколько купить)
Сигнал даёт направление, но не размер. Vol-targeting превращает
позицию 0/1 в размер, обратный волатильности инструмента. Подробная
механика — в Части 2 ниже (это твой вопрос).

### Уровень 3. Ансамбль (для MR-ноги)
Вместо одного MR-варианта берётся среднее нескольких. Это снижает
зависимость от удачного выбора одного варианта. Какие именно варианты
усреднять — решено анализом вклада (LOO), см. таблицы.

### Уровень 4. Защитный гейт (опционально)
`vol_percentile_gate` блокирует вход, когда волатильность инструмента
выходит за 90-й перцентиль своей истории — защита от структурных
коллапсов (нефть-2020). Работает на сырье, бесполезен на акциях
(проверено).

### Уровень 5. Портфель из ног (vol-parity)
Тренд-нога и MR-нога объединяются с весами, обратными их собственной
волатильности. Так каждая нога вносит равный риск, а не равный доллар.
Ключ: корреляция ног ≈ 0 (0.04 сырьё, 0.30 акции) — они почти
независимы, поэтому просадки не совпадают.

### Уровень 6. Портфельное плечо (vol-targeting второго уровня)
Финальный множитель на весь комбинированный P&L, чтобы довести общую
волатильность портфеля до желаемого уровня риска. Sweep находит
максимум в рамках DD<40%.

## Часть 2. Vol-targeting — как именно реализован (твой вопрос)

Короткий ответ: **база та же, что в ранних проектах (size =
target_vol / realized_vol), но добавлен один нетривиальный слой —
буфер ребалансировки.** Разбор формулы по шагам.

### Базовая формула (как в ранних проектах)
```
realized_vol = std(дневных доходностей за 30 баров) × √(баров в году)
size = target_vol / realized_vol,  обрезано сверху по max_leverage
```
Смысл: спокойный инструмент (низкая realized_vol) → размер больше;
бешеный → меньше. Цель — держать вклад инструмента в риск постоянным.
Аннуализация через √(баров в году) — одна формула работает и на
дневках (252), и на H4 (~1512).

### Что добавлено сверху: буфер ребалансировки
Проблема простой формулы: `realized_vol` пересчитывается каждый бар и
чуть дрожит, значит и `size` дрожит каждый бар. А каждое изменение
размера — это сделка, значит издержки. На дробных позициях это
«смерть от тысячи порезов» — множество мелких платных ребалансировок.

Решение (аудит 2026-07): применённый размер обновляется до нового
**только когда расхождение превышает buffer** (по умолчанию 10%). Пока
новый расчётный размер в пределах ±10% от текущего — позиция не
трогается. Это убирает микро-дрожание, оставляя реакцию на реальные
сдвиги волатильности.

Технически это вводит **состояние** (размер зависит от предыдущего
применённого размера) — то есть простой векторной формулой уже не
считается, нужен проход по барам в цикле. Look-ahead при этом нет:
размер бара t зависит только от прошлого (волатильность до t и
применённый размер на t−1).

### Важное свойство
Vol-targeting **не создаёт прибыль** — он масштабирует и риск, и
доходность одним множителем. Это подтвердилось на всех прогонах: Sharpe
не меняется от уровня target_vol. Vol-targeting нужен, чтобы (а) держать
риск инструментов сопоставимым и (б) дать точку приложения плеча.

### Два уровня vol-targeting в системе
1. **На инструменте** (`vol_target_size`, суффикс `_vt`) — с буфером,
   описан выше.
2. **На портфеле** (`portfolio_vol_target`) — доводит волу всего комбо
   до цели, с потолком плеча и (опционально) вычетом стоимости
   фондирования. Без буфера (ребаланс раз в день на уровне множителя).

## Часть 2b. MR-алгоритмы и их математика (детально)

Все MR-варианты растут из одного базового механизма (bb_rsi) и
отличаются ОДНИМ элементом: мерой растяжения, выходом или фильтром.
Это осознанный приём — варьируем математику, а не параметры, чтобы
ошибки вариантов были в разных местах (основа для ансамбля).

### Базовый механизм (общий для всех)
Вход в лонг, когда инструмент перепродан по ДВУМ независимым мерам:
1. **Полосы Боллинджера**: `SMA(20) ± 2·std(20)`. Цена ниже нижней
   полосы = растяжение вниз более чем на 2 стандартных отклонения.
2. **RSI (Relative Strength Index)**, окно 14: осциллятор [0,100],
   меряющий отношение средних приростов к средним падениям. RSI < 30 =
   классический порог перепроданности.
Вход = обе меры сработали. Базовый выход = RSI вернулся выше 50.

### Мера растяжения: два способа
- **Bollinger (std-based)**: полосы = SMA ± k·std. Проблема: std
  раздувается одним выбросом — шоковый бар расширяет полосы на всё окно
  и глушит сигнал.
- **Keltner (ATR-based)**: полосы = EMA ± k·ATR, где ATR (Average True
  Range) по Уайлдеру — сглаженная мера дневного размаха. ATR устойчивее
  к единичным шокам, чем std. Это ключевое отличие mr_keltner.
- **Z-score** (используется как непрерывная мера в части вариантов):
  `z = (close − SMA) / std`. То же, что полосы, но непрерывно: даёт не
  «ниже/выше», а «на сколько сигм» — можно масштабировать вход.

### Четыре базовых MR-варианта (кирпичи ансамблей)

**mr_atr_stop** — база + жёсткий ATR-стоп на выход.
Математика выхода: позиция закрывается, если `low < entry_price −
2·ATR(на баре входа)`. Ключевой момент — ATR **фиксируется на баре
входа** (не плавает), иначе стоп «убегает». Зачем: RSI-выход базового
bb_rsi — НЕ риск-стоп. Пока RSI не восстановился, позиция тонет без
ограничения (это и был провал Tesla −53%). ATR-стоп обрезает левый
хвост жёстко.

**mr_time_stop** — база + выход по сроку.
Математика: позиция закрывается через `max_hold = 10` баров, если
отскок не случился. Обоснование — у mean-reversion есть half-life
(период полураспада отклонения). Если за 10 баров реверсия не
произошла, гипотеза «временная перепроданность» мертва — это уже
падающий тренд, а не откат. Классика литературы MR.

**mr_keltner** — база, но растяжение по ATR-каналу вместо std-полос.
Математика входа: `close < EMA(20) − 2·ATR(20)` вместо нижней полосы
Боллинджера. Всё остальное как база. Смысл — меньше ложных
«перепроданностей» после единичных шоков (std ими раздувается, ATR
нет). На практике оказался сильнейшим одиночным MR-вариантом.

**mr_confirm** — база + вход только по ПОДТВЕРЖДЕНИЮ разворота.
Математика входа: условия базы (Боллинджер+RSI) выполнены вчера ИЛИ
сегодня, И `close_сегодня > close_вчера` (цена начала расти). Базовый
bb_rsi ловит падающий нож — входит, пока цена ещё валится. confirm
ждёт разворотного бара: платит худшей ценой входа за то, что реже
«ловит и едет ниже». Именно поэтому он защитен в коллапсах (CL-2020):
в свободном падении подтверждения нет, вход блокирован.

### Как из кирпичей собраны ансамбли
Ансамбль = среднее позиций членов (каждый даёт 0/1, среднее даёт
[0,1] — доля членов «в рынке»). Это мягкий сайзинг по согласию:
согласны все 4 → полная позиция, согласны 2 → половина.
- **mr_ens** = (atr_stop + time_stop + keltner + confirm) / 4
- **mr_kelt_confirm** = (keltner + confirm) / 2

Почему на СЫРЬЕ выбрана пара, а на АКЦИЯХ полный ансамбль — см. Часть 5
(LOO-анализ): на сырье atr/time дублируют keltner (корр 0.9), а confirm
декоррелирован (0.3) и добавляет защиту; на акциях confirm тоже
скоррелирован (0.68), разделять нечего.

## Часть 3. Что такое каждый рабочий sleeve

**Важно: sleeve бывают одиночные (одна нога) и комбо (пара ног).**
Торгуемый портфель — это ПАРА, одиночные ноги — её компоненты.

### donchian_vt (одиночная нога, тренд, сырьё)
Пробой канала Дончиана (цена выше максимума за N баров) + EMA-фильтр
макротренда + встроенный vol-targeting. Суффикс `_vt` = vol-targeting
уже внутри. Это НЕ «голый Дончиан» — голый donchian отдельная
стратегия, которая проиграла (оборот 6-30x, Sharpe 0.28).

### ema_vt (одиночная нога, тренд, акции)
Сигнал по пересечению/наклону EMA + встроенный vol-targeting. Equity-
аналог donchian_vt. Соло даёт +92% за период (сильнейшая одиночная
нога проекта).

### mr_kelt_confirm (одиночная нога, возврат, сырьё)
Ансамбль из ДВУХ MR-вариантов: mr_keltner (вход по выходу цены за
Keltner-канал) + mr_confirm (вход только по подтверждению разворота).
Выбран через LOO+walk-forward: остальные MR-варианты дублировали
keltner (корреляция 0.9), а confirm декоррелирован (0.3) и защитен.

### mr_ens (одиночная нога, возврат, акции)
Ансамбль из ЧЕТЫРЁХ MR-вариантов (atr_stop, time_stop, keltner,
confirm). На акциях LOO показал, что члены неразделимы (все
скоррелированы 0.9, включая confirm 0.68) — выделять пару смысла нет,
полный ансамбль оправдан как есть.

### ТОРГУЕМЫЕ ПАРЫ (это и есть портфель)
- **Сырьё** = donchian_vt + mr_kelt_confirm, vol-parity, корр 0.04
- **Акции** = ema_vt + mr_ens, vol-parity, корр 0.30

## Часть 6. Закрытые треки (задокументированы с механизмом, не сдавать как рабочие)

| Трек | Механизм отклонения |
|------|---------------------|
| champion (take-profit + пирамида) | Тейк режет правый хвост тренда (Cocoa +51%→+8%); пирамида не добавляет |
| GARCH-VT | Два A/B: рефит держал плечо на входе в CL-2020; realized реактивнее |
| vol-gate на акциях | Специфичен для шоков; бесполезен против медленных просадок (Tesla-2022 −18%→−18%) |
| carry | Нужно 20-30 инструментов для факторной диверсификации |
| dual momentum (обе версии) | Концентрация, не ранжирование; разваливается без 2-3 имён |
| 1-факторная OU | 94% ложных срабатываний; сырьё двухфакторно (нужен Schwartz-Smith) |
| HMM-режимы | Дублирует vol-targeting как неявный детектор режима |
| pairs (zscore/Kalman) | Нет edge даже при нулевых издержках — структурная проблема |
| шорт-ноги | На растущих активах несут только бету, не альфу (4 слоя доказательств) |
| EMA200-фильтр | Документированный лаг |
| H4 для тренда | Издержки съедают сигнал на всех Дончиан-вариантах кроме donchian_vt |
| Bollinger %B / TTM Squeeze | Провал стабильности параметров (переобучение) |

## Часть 8. Как запускать прогоны (команды и флаги)

Три раннера покрывают всё. Ниже — универсальная команда каждого с
расшифровкой флагов, затем готовые вызовы для рабочих и альтернатив.

### Раннер 1 — walk-forward (робастность стратегии по годам)
```
python -m runners.run_walkforward --strategy <ИМЯ...> \
    --basket <commodity|equity> [--source <yf|databento>] \
    [--panel-dir data/panels/futures] --start 2019-01-01 \
    [--vt] [--sizer <realized|garch>] [--matrix]
```

### Раннер 2 — sleeves (комбо ног + плечо)
```
python -m runners.run_sleeves --sleeve <strat:basket[:vt|:garch]> \
    --sleeve <strat:basket[:vt|:garch]> [--parity] \
    [--source <yf|databento>] [--panel-dir ...] --start 2019-01-01 \
    [--port-vol 0.20 --max-port-lev 4] [--lev-sweep] \
    [--funding-rate 0.06]
```

### Раннер 3 — вклад членов ансамбля (LOO)
```
python -m runners.run_member_contribution --ensemble <mr|trend> \
    --basket <commodity|equity> [--source ...] [--panel-dir ...] \
    --start 2019-01-01 [--vt] [--target-vol 0.35]
```

### Расшифровка флагов
| Флаг | Значение |
|------|----------|
| `--strategy X Y Z` | одна или несколько стратегий из реестра (сравнить рядом) |
| `--sleeve strat:basket:vt` | нога комбо; `:vt` = realized-vol-targeting, `:garch` = GARCH-VT, без суффикса = сырой сигнал |
| `--basket commodity/equity` | корзина инструментов (сырьё / акции) |
| `--source yf/databento` | данные: yfinance (акции, дневные) / Databento (сырьё, roll-adjusted). Для databento обязателен `--panel-dir` |
| `--panel-dir PATH` | путь к parquet-панелям Databento (`data/panels/futures`) |
| `--start / --end` | период бэктеста (2019-01-01 = включая ковид) |
| `--vt` | наложить vol-targeting на позицию (size = target_vol/realized_vol + буфер) |
| `--sizer realized/garch` | оценка волы для `--vt`: rolling-std (рабочая) / GARCH (закрыт, для проверки) |
| `--target-vol 0.35` | целевая годовая вола для `--vt` |
| `--matrix` | печатать матрицу год × инструмент (детальная разбивка) |
| `--parity` | веса ног обратны их воле (trailing, без look-ahead); иначе равные |
| `--port-vol 0.20` | портфельное плечо: довести волу КОМБО до 20% (одна точка) |
| `--max-port-lev 4` | потолок портфельного плеча для `--port-vol` |
| `--lev-sweep` | перебрать сетку target_vol × кэп плеча, найти макс доход в DD<40% |
| `--funding-rate 0.06` | вычесть стоимость плеча (6%/год на заёмную часть). Для акций применимо, для фьючерсов гипотетика |

### Готовые вызовы — РАБОЧИЕ (4 sleeve, 2 пары)

```
# Сырьё: обе ноги walk-forward
python -m runners.run_walkforward --strategy donchian_vt mr_kelt_confirm \
    --basket commodity --source databento --panel-dir data/panels/futures \
    --start 2019-01-01 --vt --matrix

# Акции: обе ноги walk-forward
python -m runners.run_walkforward --strategy ema_vt mr_ens \
    --basket equity --start 2019-01-01 --vt --matrix

# Сырьё: финальное комбо + плечо
python -m runners.run_sleeves --sleeve donchian_vt:commodity \
    --sleeve mr_kelt_confirm:commodity:vt --parity --start 2019-01-01 \
    --source databento --panel-dir data/panels/futures --lev-sweep

# Акции: финальное комбо + плечо
python -m runners.run_sleeves --sleeve ema_vt:equity \
    --sleeve mr_ens:equity:vt --parity --start 2019-01-01 --lev-sweep
```

### Готовые вызовы — АЛЬТЕРНАТИВЫ

Все альтернативы гоняются той же командой walk-forward, меняется только
имя в `--strategy`. Универсальный шаблон для сырья:

```
python -m runners.run_walkforward --strategy <ИМЯ> \
    --basket commodity --source databento --panel-dir data/panels/futures \
    --start 2019-01-01 --vt --matrix
```
Подставить `<ИМЯ>`: `mr_keltner`, `mr_ens_gate`, `mr_ens`,
`mr_atr_stop`, `mr_time_stop`, `mr_confirm`, `tsmom_multi`, `ewmac`,
`trend_ens`, `kama`, `channel_pos`, `donch_multi`, `chandelier`,
`adx_donch`, `tsmom`, `donchian_vt`.

Проверить кандидата на ДВУХ источниках (правило двух источников) —
прогнать дважды, меняя `--source`:
```
# источник 1: yfinance
python -m runners.run_walkforward --strategy <ИМЯ> --basket commodity \
    --start 2019-01-01 --vt --matrix
# источник 2: databento (тот же период)
python -m runners.run_walkforward --strategy <ИМЯ> --basket commodity \
    --source databento --panel-dir data/panels/futures \
    --start 2019-01-01 --vt --matrix
```

Плечо для альтернативы (напр. keltner-соло) — через sleeves с одной
ногой:
```
python -m runners.run_sleeves --sleeve <ИМЯ>:commodity:vt --parity \
    --start 2019-01-01 --source databento \
    --panel-dir data/panels/futures --lev-sweep
```

LOO-разбор состава любого ансамбля:
```
python -m runners.run_member_contribution --ensemble mr \
    --basket <commodity|equity> [--source databento --panel-dir ...] \
    --start 2019-01-01 --vt --target-vol 0.35
```
