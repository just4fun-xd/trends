Сырьевой тренд: donchian_vt (единственная модель РОБАСТНА и на yf, и на

тестов не осталось; дальше — полнота корзины, тренд-ансамбль и
портфельная математика.

## ОТКЛОНЕНО (закрыто отрицательно, с механизмами)

- **GARCH-VT** — два независимых A/B (mr_ens-2019, donchian): Sharpe
  ниже на каждом уровне; в хвосте (CL-2020) квартальный рефит ДЕРЖАЛ
  плечо на входе в коллапс. Rolling-30 реактивнее на дневках.
- **champion (take-profit + пирамида)** — два прогона: тейк режет
  правый хвост, где живёт P&L тренда (Cocoa-24: +51% vs +8%); пирамида
  не добавляет (4step≈champion).
- **tsmom одиночный (12м)** — 71% окон на обоих источниках: один
  горизонт momentum нестабилен. Мульти-горизонт чинит (86% на yf).
- **tsmom_multi как САМОСТОЯТЕЛЬНЫЙ чемпион** — на узкой DB-корзине
  (7) деградировал до 71%, разброс 26.4%: высокая бета-к-тренду без
  фильтра ложных пробоев. Кандидат в АНСАМБЛЬ, не в чемпионы.
- Ранее: carry, dual momentum (оба вида), 1-фактор OU, HMM, EMA200,
  pairs, шорт-ноги, H4 для тренда, BB %B/TTM.

## ОЧЕРЕДЬ (пересобрана по данным, не по теории)

**Блок A — полнота данных (день):**
1. Дособрать панель до 12: `python -m scripts.fetch_databento_futures
   --interval 1d --start 2019-01-01 --end 2026-01-01` (дефолт теперь из
   конфига; кэш докачает только ZS ZL ZM PA PL).
2. Перегнать mr_ens_gate и конкурс тренда на полной DB-корзине.
3. **Sleeves на Databento** — комбо-прогон шёл на yf (в логе «(yf,
   1d)»): добавить `--source databento --panel-dir data/panels/futures`.

**Блок B — тренд-ансамбль (2-3 дня):**
4. `trend_ens` = среднее сырых ewmac + tsmom_multi + donch_multi
   (по образцу mr_ens; donchian_vt НЕ внутрь — двойной VT). Сравнить с
   donchian_vt на обоих источниках. Данные за: три модели 86% на yf,
   ewmac 86% и на DB, а ошибки у них в разные годы (2023: donch_multi
   −, ewmac −, kama +16 Gold).
5. Carver forecast scaling + FDM поверх trend_ens (правильный масштаб
   комбинированного прогноза с учётом корреляции ног).

**Блок C — портфельная математика (неделя):**
6. 4-sleeve портфель одним прогоном: donchian_vt + mr_ens_gate
   (commodity) + ema_vt + mr_ens (equity), vol-parity → затем
   Ledoit-Wolf shrinkage весов (при 4 ногах уже осмыслен).
7. Block bootstrap P&L-кривых: доверительные интервалы Sharpe — «trend_ens
   лучше donchian_vt» должно стать проверяемым утверждением, а не
   сравнением точечных чисел (+5.6 vs +4.4 на 7 окнах — пока шум).

**Блок D — исследовательские (после A-C):**
8. Гейт: sweep порога 0.85/0.90/0.95 + вариант entry-only (блокировать
   вход, но не выбрасывать из позиции) + EVT/POT-порог вместо
   эмпирического перцентиля.
9. mr_ens_gate на EQUITY (гейт должен закрывать Tesla-хвост так же,
   как CL-2020; corr equity-пары 0.22 может упасть).
10. Meta-labeling (triple barrier + логрегрессия на numpy) для
    donchian_vt: пропускать/нет сделку.
11. Schwartz-Smith 2-фактор (Kalman): OU-остаток → MR-нога.
12. Расширение breadth: softs через ICE-фид (KC/CC/SB/CT вне GLBX),
    FX/rates futures — доход тренда ~√N независимых рынков.

## НОВЫЕ ИДЕИ (добавлены по итогам валидации)

- **Правило двух источников** (в дисциплину проекта): стратегия
  «выпускается» только при РОБАСТНА на yfinance И Databento. tsmom_multi
  — первый пойманный этим правилом кандидат.
- **Портфельный vol-gate**: тот же перцентильный механизм на СРЕДНЕЙ
  воле корзины — общий тормоз всех ног в системном кризисе (когда
  корреляции летят к 1 и посерийная диверсификация исчезает).
- **Корреляционные веса внутри sleeve**: inverse-vol × (1 − ср.
  корреляция с корзиной); на 7-инструментной DB-корзине энергия
  доминирует риск-бюджет.
- **Диагностика вклада гейта**: сколько баров закрыто гейтом по
  инструменту/году + P&L заблокированных баров — «цена страховки»
  одним числом для отчёта.
