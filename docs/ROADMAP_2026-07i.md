## ПРИОРИТЕТ 1 — Зафиксировать боевые корзины в конфиг

Сейчас корзины набираются руками (`@CL,ZS,SI...`). Вынести в
core/config как именованные константы, чтобы не подгонять каждый раз:
```
DONCH_CORE_COMM = [CL, ZS, SI, ZL, HG, GC]
MRLV_CORE_COMM  = [CL, NG, SI, ZW, ZC, GC]
ICHI_CRYPTO / DONCH_CRYPTO = минус [Litecoin, Cosmos]
```
Плюс подтвердить сырьевой v2 на yf (второй источник) тем же @include.
Пересмотр корзин — раз в квартал, не чаще (иначе подгонка).

## ПРИОРИТЕТ 2 — Кросс-классовый HRP-портфель (ДЛЯ АЛЕКСАНДРА)

Первый портфель, где HRP встретит НАСТОЯЩИЕ кластеры:
```
--sleeve donchian_vt:commodity:vt@DONCH_CORE \
--sleeve mr_lowvol:commodity:vt@MRLV_CORE \
--sleeve donchian_vt:crypto:vt (+ ichimoku ансамбль) \
--hrp
```
Кластеры: сырьё-тренд + сырьё-MR (некоррелированы) vs крипта
(отдельная вселенная). HRP распределит риск между ветвями, не активами.
Цель: единая портфельная строка — Sharpe, годовая, DD, корр с S&P.
Проблема: сырьё daily vs крипта H4 — разные bpy/календари. Решить:
либо крипту агрегировать в daily, либо считать веса на дневных P&L
(ресемпл H4->1d для матрицы корреляций).
