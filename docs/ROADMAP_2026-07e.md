# ROADMAP 2026-07e — после batch-прогона и вердикта Александра

## Контекст приоритизации

Вердикт Александра («bonds would give you better ROI») сместил фокус:
главная проблема — не робастность (она доказана), а **тонкая сырая
доходность** (3.4%/год < rf). Плечо как ответ отклонено (funding на
марже съедает всё; для фьючерсов вопрос — initial margin, не займ).
Новый приоритет №0 — доходность без плеча.

## БЛОК 0 — Прибыль без плеча (немедленно)

0.1 **rf в Sharpe** (`--rf 0.045` в metrics) — закрыть дыру
    методологии до того, как её снова найдёт Александр. [~30 строк]
0.2 **Per-instrument target_vol sweep 15→40%** на финальных парах:
    для фьючерсов это размер в рамках обеспечения. Ожидание: портфель
    ~6-10% волы, ~8-12% доходности при том же Sharpe, DD-проверка
    sweep'ом. [конфиг + прогон]
0.3 **Collateral yield в отчётность**: строка «+rf на обеспечение»
    для фьючерсного счёта (portable alpha фрейм). [метрики/отчёт]
0.4 **Ре-аудит шорт-ноги сырья**: закрытие (−71%) могло быть
    загрязнено raw-vs-VT (как ewmac/trend_ens). One-shot, два
    источника, единообразный VT. [прогон]

## БЛОК A — Принятые к реализации (№6, №1, №2, №8)

A.1 **Block bootstrap** (stationary bootstrap Политиса-Романо):
    CI на Sharpe и на разность Sharpe. Первый вопрос к нему —
    kama (+4.4) vs donchian_vt (+3.4): значимо или шум.
A.2 **Carver forecast scaling + FDM**: непрерывные прогнозы
    (ewmac-пары + kama + donch_multi), клип ±20, FDM=1/√(w'Ρw).
    Двойная роль: рост Sharpe (диверсификация прогнозов) И рост
    использования волы (FDM>1, меньше кэша). Данные за: все члены
    двухисточниковые после реабилитации.
A.3 **Hurst-аллокатор** (R/S + variance-ratio Ло-МакКинлея):
    вес тренд/MR-ноги на инструменте по его H. Не HMM (свойство
    инструмента, не режим времени).
A.4 **Volume z-score подтверждение**: пробой валиден при z(volume)>1.
    Volume уже в Bars (опционален) и в Databento-панелях —
    panel_volume.parquet не используется вообще.

## БЛОК C — Портфельная математика

C.2 Онлайн-веса Hedge/multiplicative-weights (режимность через веса,

C.3 Breadth: FX/rates/крипто-корзины тем же конвейером (двухэтапно:

## БЛОК D — Исследовательская очередь
